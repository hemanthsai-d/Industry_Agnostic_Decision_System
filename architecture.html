Components

What Each Piece Does

Every component has a specific job. Here's what each one does in plain language.

⚡

FastAPI Application Server

What it does: The main brain of the system. It receives customer messages via HTTP, processes them through all 8 pipeline stages, and returns a decision.

In real life: Like the front desk of a hospital — it takes in patients (messages), runs diagnostics (pipeline), and decides the treatment path (decision).

PORT: 8000 ENDPOINTS: /v1/assist/decide, /v1/assist/feedback, /health THROUGHPUT: 120+ requests/min per tenant
🧠

Model Server

What it does: A separate service that runs the AI classification models. It receives text and returns probabilities for 30 different intent categories.

In real life: Like a specialist doctor — the main system sends it a case description, and it returns a diagnosis with confidence percentages.

PORT: 9000 MODELS: Routing (30 intents) + Escalation (binary) BACKENDS: Artifact (JSON) → Heuristic fallback
🐘

PostgreSQL + pgvector

What it does: Stores everything — customer interactions, knowledge base articles, AI decisions, model evaluation data. pgvector adds AI-powered search capabilities.

In real life: The filing cabinet and library combined — stores all records AND can do smart searches to find related documents.

VERSION: PostgreSQL 16 EXTENSIONS: pgvector (vector similarity search) TABLES: 20+ (decisions, handoffs, feedback, chunks, metrics)
🔴

Redis Cache

What it does: Provides rate limiting (prevents abuse) and caching (speeds up repeated lookups). Acts as a fast memory layer.

In real life: Like a bouncer at a club — it controls how many people get in per minute, and remembers frequent visitors.

VERSION: Redis 7 Alpine USES: Rate limiting, idempotency cache, session store LIMIT: 120 req/min per tenant, 60 req/min per user
⚖️

Open Policy Agent (OPA)

What it does: Enforces business rules. Even if the AI is confident, OPA can override based on rules — like "always escalate legal threats" or "block low-confidence responses."

In real life: The compliance officer — reviews every decision against company policy before it goes out.

VERSION: OPA 0.69.0 LANGUAGE: Rego (policy-as-code) RULES: High-risk term detection, confidence threshold, escalation limits
🔭

Observability Stack

What it does: Monitors everything in real-time. Prometheus collects numbers, Grafana shows graphs, Alertmanager pages people when something goes wrong, Jaeger traces individual requests.

In real life: The hospital's vital signs monitors — always watching, always alerting if something looks off.

PROMETHEUS: Collects 25+ metrics every 15 seconds GRAFANA: 5+ dashboard panels with real-time graphs ALERTMANAGER: 14 alert rules, 4 severity-based routes JAEGER: Distributed tracing for request debugging

Data Flow

How Data Moves Through the System

Follow a customer message from arrival to final decision.

📩 Step 1: Customer Message Arrives

A customer writes: "I was charged twice for my subscription last month". This arrives via the /v1/assist/decide endpoint with tenant context.

🔒 Step 2: PII Redaction (~2ms)

The system scans for 13 types of personal information (credit cards, emails, SSNs, etc.) and masks them before any processing begins. Privacy first.

🛡️ Step 3: Prompt Injection Defense (~3ms)

A 3-layer scanner checks for manipulation attempts: 18 regex patterns, role marker detection, and character entropy scoring. Malicious inputs are blocked.

🔎 Step 4: Hybrid Evidence Retrieval (~80ms)

The system searches the knowledge base using both keyword matching (PostgreSQL FTS) and AI similarity (pgvector). Results are merged using Reciprocal Rank Fusion, then reranked and deduplicated.

🧠 Step 5: AI Intent Classification (~15ms)

The model server analyzes the message and returns probabilities across 30 intent categories. In this case: refund_duplicate_charge: 62.8%, account_access_recovery: 18.0%, etc.

📏 Step 6: 5-Signal Confidence Scoring (~5ms)

Five separate signals are combined: route confidence (45%), evidence quality (25%), escalation risk (20%), OOD detection (−7%), contradiction (−3%). The final score determines the decision.

⚖️ Step 7: Policy Gate Check (~2ms)

OPA evaluates business rules: Is this a high-risk term (legal, fraud)? Is confidence above threshold? Is escalation probability acceptable? Rules can override AI.

✅ Step 8: Decision Output

The system returns RECOMMEND with a draft response, evidence citations, confidence breakdown, and trace ID for debugging. Total time: ~107ms.

Data Model

Database Schema (10 Migrations)

The database evolves through sequential migrations, each adding new capabilities.

📋 Migration 1–3: Core Tables

Core decision tables — stores every decision made (tenants, decisions, evidence used). Inference audit — logs model predictions for evaluation. Feedback events — captures human corrections to improve the model.

📊 Migration 4–6: ML Ops

Model evaluation — daily accuracy and recall metrics per model variant. Rollout evidence — canary promotion gate pass/fail records. Sample size gates — minimum data requirements before promoting models.

📈 Migration 7–8: Business KPIs

Business scorecard — adoption rate, accuracy, handling time reduction KPIs. Operational controls — secret rotation, access reviews, compliance tracking.

🔐 Migration 9–10: Security

Intent taxonomy & PII audit — tracks PII detections and intent classification history. Tenant RLS & data retention — row-level security and automatic data cleanup policies.
