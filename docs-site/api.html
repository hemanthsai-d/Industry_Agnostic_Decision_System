Core Endpoint

POST /v1/assist/decide

The main endpoint. Submit a customer message and get back an AI-powered decision with confidence scores, intent classification, and recommended action.

📤 Request Body assist:decide
FieldTypeDescription
issue_text REQUIRED string The customer's message
customer_tier string "basic", "premium", "enterprise"
channel string "chat", "email", "phone", "web"
session_id string Optional session identifier for context tracking
POST /v1/assist/decide Content-Type: application/json Authorization: Bearer eyJhbG... { "issue_text": "I was charged twice for my subscription last month", "customer_tier": "premium", "channel": "chat" }
📥 Response (200 OK)
{ "request_id": "abc123-def456", "decision": "auto_respond", "intent": "billing_dispute", "confidence": 0.87, "confidence_signals": { "route_confidence": 0.92, "evidence_quality": 0.85, "escalation_risk": 0.12, "ood_score": 0.03, "contradiction_flag": 0.01 }, "response": "I understand you were charged twice. Let me look into your billing for last month...", "evidence_chunks": [ { "text": "For duplicate charges...", "score": 0.91 } ], "policy_override": null, "processing_time_ms": 142 }
✅

"auto_respond"

AI is confident — sends the response directly to the customer without human intervention.

🤷

"abstain"

AI is unsure — routes to a human agent with all the gathered evidence attached.

🚨

"escalate"

High-risk content detected — immediately flags for senior agent review.

Feedback Loop

POST /v1/assist/feedback

Human agents submit feedback on AI decisions. This data is used to measure accuracy and improve the models over time.

📤 Request assist:feedback
POST /v1/assist/feedback Authorization: Bearer eyJhbG... { "request_id": "abc123-def456", "verdict": "correct", "corrected_intent": null, "agent_notes": "AI decision was accurate" }
FieldValues
verdict"correct", "incorrect", "partial"
corrected_intentThe correct intent if AI was wrong
agent_notesOptional free-text notes
📥 Response (200 OK)
{ "status": "recorded", "feedback_id": "fb-789", "routing_accuracy_7d": 0.847 }

The response includes the rolling 7-day routing accuracy so agents can see the AI's overall performance improving over time.

Escalation Queue

Handoff Endpoints

When the AI escalates or abstains, the case enters the handoff queue. Human agents use these endpoints to manage escalated cases.

GET /v1/assist/handoffs assist:handoff:read

Lists all pending handoffs (escalated cases waiting for human review).

GET /v1/assist/handoffs?status=pending&limit=20 # Response: { "handoffs": [ { "id": "hoff-001", "request_id": "abc123-def456", "reason": "policy_high_risk", "intent": "billing_dispute", "confidence": 0.42, "customer_tier": "enterprise", "created_at": "2026-02-17T10:30:00Z", "status": "pending" } ], "total": 3, "page": 1 }
PATCH /v1/assist/handoffs/{id} assist:handoff:update

Resolve or reassign a handoff case.

PATCH /v1/assist/handoffs/hoff-001 { "status": "resolved", "resolution_notes": "Issued full refund for duplicate charge", "resolved_by": "agent_smith" } # Response: { "id": "hoff-001", "status": "resolved", "resolved_at": "2026-02-17T10:45:00Z", "time_to_resolve_min": 15 }

Error Handling

Error Responses

All errors return JSON with a consistent structure. Use the error code to handle specific cases.

⚠️ Error Codes
HTTP StatusError CodeMeaningWhat To Do
400 VALIDATION_ERROR Invalid request body Check required fields and types
400 INJECTION_DETECTED Prompt injection blocked Do not retry — input is flagged
401 AUTH_REQUIRED Missing or expired JWT Generate a new token
403 FORBIDDEN Token lacks required permission Request appropriate role
429 RATE_LIMITED Too many requests Wait for Retry-After header value
500 INTERNAL_ERROR Unexpected server error Retry with exponential backoff
# Error response format: { "error": { "code": "RATE_LIMITED", "message": "Rate limit exceeded for tenant acme-corp", "request_id": "req-abc123", "retry_after_seconds": 30 } }
Getting Started

Authentication

All API calls (except health checks) require a JWT Bearer token.

🔑 Generate a Token
# Using the token generator script: python scripts/generate_token.py \ --tenant acme-corp \ --role agent \ --permissions assist:decide,assist:feedback # Output: eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...
📡 Use the Token
# Include in every request: curl -X POST http://localhost:8000/v1/assist/decide \ -H "Content-Type: application/json" \ -H "Authorization: Bearer eyJhbG..." \ -d '{ "issue_text": "My order hasn't arrived" }'
